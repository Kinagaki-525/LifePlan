# ライフプランシミュレーター仕様書

## 概要

本ドキュメントは、`rennsyu` の ASP.NET Core MVC アプリに追加するライフプランシミュレーターの実装前提を整理した仕様書である。  
入力フォームで家族構成、収入、貯蓄、毎月の支出、ローン、ライフイベントを入力し、年ごとの収入・支出・預貯金残高をグラフと表で確認できる Web アプリを対象とする。

本仕様は v1 を対象とし、永続化や高度な資産運用計算は含めない。

## 目的とゴール

- ユーザーが現在の家計条件と将来の大きな支出を入力できること
- 現在年から本人 90 歳までの年次推移を確認できること
- 年ごとの収入、支出、預貯金残高を一覧とグラフで把握できること
- ASP.NET Core MVC の既存構成を崩さず、責務分離を意識した実装にすること
- 次の実装フェーズで迷わないよう、入力項目、計算ルール、表示仕様を明確にすること

## 画面構成

画面は `Home/Index` の 1 画面完結とする。入力エリアと結果エリアを同一画面に配置する。

### 1. 入力エリア

以下のセクションを上から順に表示する。

1. 家族構成
2. 収入
3. 貯蓄
4. 毎月の支出
5. ローン
6. ライフイベント
7. シミュレーション実行ボタン

### 2. 結果エリア

シミュレーション実行後に以下を表示する。

1. サマリー
2. 年次推移グラフ
3. 年次一覧表

初期表示時は結果エリアを空、またはプレースホルダー表示とする。

## 入力仕様

金額入力の単位は原則「万円」とする。  
内部計算では円に換算して扱う。

### 家族構成

- 本人年齢
  - 必須
  - 整数
  - 0 以上 90 以下を想定
- 配偶者年齢
  - 任意
  - 未入力時は配偶者なしとして扱う
  - 入力時は整数
- 子ども
  - 可変件数
  - 0 件以上
  - 各行に子どもの現在年齢を入力する
  - 負の値を許容し、将来生まれる予定の子どもも入力可能とする
  - 入力時は整数
  - 子ども専用の自動学費計算は行わない

### 収入

本人・配偶者それぞれについて以下を持つ。

- 現在年収
  - 任意
  - 未入力時は 0 として扱う
  - 単位は万円
- 退職年齢
  - 任意
  - 未入力時は退職金なし、将来収入なしとして扱う
  - 整数
- 退職金
  - 任意
  - 未入力時は 0 として扱う
  - 単位は万円

補足:

- 本人の収入条件は入力前提とする
- 配偶者の収入条件は任意とする
- 年金、昇給率、副収入、賞与の詳細入力は v1 では扱わない

### 貯蓄

- 現在の貯蓄額
  - 必須
  - 単位は万円
  - シミュレーション開始時点の初期残高として使う

### 毎月の支出

- 現在の毎月支出
  - 必須
  - 単位は万円
- 年増加率
  - 必須
  - 単位は %
  - 毎年の通常支出に対して乗算で反映する

補足:

- 毎月支出には生活費、住居費、保険料などの固定費・変動費をまとめて含める
- ローン返済はこの項目に含めず、ローン入力で別管理する
- 住宅購入や自動車購入の頭金など、大きな一時支出はライフイベントで扱う

### ローン

可変件数の行追加式とする。各行は以下の項目を持つ。

- 名称
  - 任意
  - 例: 住宅ローン、マイカーローン、教育ローン
- 借入金額
  - 必須
  - 単位は万円
- 年利
  - 必須
  - 単位は %
- 返済開始年月
  - 必須
  - 年月入力
- 返済終了年月
  - 必須
  - 年月入力

補足:

- ローンは継続的な支出として扱う
- 各ローンは返済開始年月から返済終了年月まで、毎月一定額を支払う前提とする
- 月々の返済額は元利均等返済で算出する
- 繰上返済、ボーナス払い、変動金利、返済額変更は v1 では扱わない

### ライフイベント

可変件数の行追加式とする。各行は以下の項目を持つ。

- 名称
  - 任意
  - 例: 車購入、住宅頭金、大学入学
- 発生年齢
  - 必須
  - 本人年齢を基準にした年齢
  - 整数
- 金額
  - 必須
  - 単位は万円

補足:

- ライフイベントは単発の支出として扱う
- 学費はライフイベントとして入力する
- 子どもの大学入学なども自動計算せず、必要なタイミングと金額を手動入力する
- ライフイベントは複数件追加できる

## 計算仕様

### 試算期間

- 開始年はシミュレーション実行時の現在年とする
- 終了年は本人が 90 歳になる年とする
- 年単位でシミュレーションする

### 年齢計算

- 各年の本人年齢は `本人の現在年齢 + 経過年数`
- 各年の配偶者年齢は `配偶者の現在年齢 + 経過年数`
- 子どもの年齢は負の値を許容し、`-1` は 1 年後に生まれる予定、`-2` は 2 年後に生まれる予定として解釈する
- ライフイベントは、その年の本人年齢が発生年齢と一致した年に反映する

### 収入計算

- 本人年収
  - 現在年から、本人年齢が退職年齢未満の年まで毎年加算する
- 本人退職金
  - 本人年齢が退職年齢と一致した年に 1 回だけ加算する
- 配偶者年収
  - 配偶者情報がある場合のみ、配偶者年齢が退職年齢未満の年まで毎年加算する
- 配偶者退職金
  - 配偶者年齢が退職年齢と一致した年に 1 回だけ加算する
- 退職後の継続収入
  - v1 では加算しない

### 支出計算

- 通常支出
  - `現在の毎月支出 × 12` を基準にする
  - 2 年目以降は年増加率を反映する
  - 計算式は `基準年支出 × (1 + 年増加率) ^ 経過年数`
- ローン返済支出
  - 各ローンについて元利均等返済の月額を算出する
  - 月額返済額は、借入金額、年利、返済開始年月、返済終了年月から決める
  - 当年に返済対象となる月数分だけ年支出に加算する
  - 返済開始年と返済終了年は、対象月数に応じて年額を按分する
- ライフイベント支出
  - 当年の本人年齢と発生年齢が一致するイベントを、その年の支出に加算する

ローン月額の算出は以下の前提とする。

- 返済回数は返済開始年月から返済終了年月までの月数で求める
- 月利は `年利 ÷ 12` とする
- 年利が 0% の場合は `借入金額 ÷ 返済回数` を月額返済額とする
- 年利が 0% 以外の場合は元利均等返済の一般式で月額返済額を算出する
- 内部計算は円単位で行う

### 預貯金残高計算

- 初年度開始時点の残高は現在の貯蓄額とする
- 各年の年末残高は以下で算出する

`年末残高 = 前年末残高 + 年収入 - 年支出`

補足:

- 投資利回り、税引き、インフレ調整後資産、借入残高などは v1 では扱わない

## 表示仕様

### サマリー

少なくとも以下を表示対象とする。

- シミュレーション開始年
- シミュレーション終了年
- 最終年の預貯金残高
- 預貯金残高が最も少ない年とその金額
- ローン返済が終了する最終年

### 年次推移グラフ

以下の 3 系列を同じグラフ内に表示する。

- 年収入
- 年支出
- 年末預貯金残高

要件:

- 横軸は年
- 縦軸は金額
- 表と同じデータソースを使う
- 描画は共通の `site.js` ではなく、専用ファイル `wwwroot/js/life-plan-chart.js` で行う

### 年次一覧表

各年について少なくとも以下の列を持つ。

- 年
- 本人年齢
- 配偶者年齢
- 年収入
- 年支出
- 年末預貯金残高

必要に応じて以下を追加してよい。

- ローン返済額
- ライフイベントの有無または名称
- 子ども人数

## 実装方針

### アーキテクチャ方針

- `Controllers`
  - HTTP リクエストの受付、入力の受け取り、View 返却に集中する
- `ViewModels`
  - 画面入力と画面表示専用の型を持つ
- `Application/Services`
  - 入力 ViewModel 相当のデータを計算用モデルに変換し、シミュレーションを実行し、結果表示用モデルを組み立てる
- `Application/Interfaces`
  - 試算サービス契約を置く
- `Domain/Entities`
  - 家族構成、収入条件、支出条件、ローン、ライフイベント、年次結果などの計算対象データを置く
- `Domain/Logic`
  - 年次シミュレーションの純粋計算を置く

### 主要型

- `LifePlanPageViewModel`
  - 画面全体の入力値、結果一覧、サマリー、結果表示フラグを持つ
- `FamilyInputViewModel`
  - 本人年齢、配偶者年齢、子ども一覧を持つ
- `IncomeInputViewModel`
  - 本人・配偶者の収入条件を持つ
- `ChildInputViewModel`
  - 子どもの現在年齢を持つ
- `LifeEventInputViewModel`
  - 名称、発生年齢、金額を持つ
- `LoanInputViewModel`
  - 名称、借入金額、年利、返済開始年月、返済終了年月を持つ
- `YearlyPlanResultViewModel`
  - 年別の表示用結果を持つ

### 主要インターフェース

- `ILifePlanSimulationService`
  - 画面入力を受け取り、サマリーと年次結果を返す

### 主要画面アクション

- `GET /Home/Index`
  - 初期値または空の入力フォームを表示する
- `POST /Home/Index`
  - 入力値を受け取り、バリデーション後に試算結果を同画面へ返す

## バリデーション方針

- 本人年齢、現在の貯蓄額、現在の毎月支出、年増加率は必須
- 金額項目は負数を不可とする
- 本人年齢、配偶者年齢、ライフイベント発生年齢は負数を不可とする
- 子どもの年齢だけは負数を許容する
- ローンの借入金額、年利、返済開始年月、返済終了年月は必須
- ローンの返済終了年月は返済開始年月以降でなければならない
- 配偶者関連は未入力を許容する
- 子どもは 0 件を許容する
- ローンは 0 件を許容する
- ライフイベントは 0 件を許容する
- ライフイベント行が存在する場合、発生年齢と金額は入力必須とする
- ローン行が存在する場合、借入金額、年利、返済開始年月、返済終了年月は入力必須とする
- 不正入力時は同一画面上に検証メッセージを表示する

## 非対応項目

以下は v1 の対象外とする。

- 年金収入
- 昇給率や賞与率の詳細設定
- 資産運用利回り
- 税金や社会保険料の控除計算
- 繰上返済、ボーナス払い、変動金利を含む高度なローン返済シミュレーション
- 月単位シミュレーション
- データ保存機能
- ユーザー認証

## 受け入れ条件

- `Home/Index` でライフプランシミュレーター画面が表示される
- 家族構成、収入、貯蓄、毎月の支出、ローン、ライフイベントを入力できる
- 本人のみの入力でもシミュレーションできる
- 配偶者あり、子ども複数、ローン複数、ライフイベント複数でもシミュレーションできる
- 退職年に退職金が 1 回だけ反映される
- 通常支出に年増加率が反映される
- ローン返済が返済対象期間の年だけ継続支出として加算される
- 発生年齢が一致した年だけライフイベント支出が加算される
- 年次一覧表とグラフが同じ計算結果を表示する
- 任意項目が未入力でも例外にならない
- 実装後に `dotnet build` が通る

## 前提・補足

- `AGENTS.md` はリポジトリルートに維持する
- 本仕様書の保存先は `docs/life-plan-simulator-spec.md` とする
- 参考サイトは UI 体験の方向性のみ参照し、厳密な再現は行わない
- 今後、年金や投資利回り、繰上返済などを追加する場合は v2 以降の拡張として扱う
